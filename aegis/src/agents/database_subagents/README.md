# Database Subagents (`iris/src/agents/database_subagents/`)

This directory contains the specialized agents (subagents) responsible for interacting with specific internal and external data sources used by the IRIS system. It also includes the `database_router.py` module, which manages the concurrent execution of these subagents based on the plan generated by the `agent_planner`.

## Purpose

Each subagent is tailored to query a particular database or data source, understanding its specific schema, query language, or API interaction patterns. This modular approach allows for easier maintenance and extension of data source integrations.

## Key Components

*   **`database_router.py`**: This module receives the query plan from the `agent_planner`. It identifies the relevant subagents, invokes them concurrently (often using threading or asynchronous operations), collects their results, and passes the aggregated information to the `agent_summarizer`.
*   **Subagent Directories (`external_*`/`internal_*`)**: Each subdirectory represents a connection to a specific data source.
    *   **External Sources**: Connect to third-party databases or APIs (e.g., `external_ey`, `external_iasb`, `external_kpmg`, `external_pwc`).
    *   **Internal Sources**: Connect to internal RBC databases or knowledge stores (e.g., `internal_capm`, `internal_cheatsheets`, `internal_compliance`, `internal_esg`, `internal_ext_reporting_and_disclosure`, `internal_global_finance_standards`, `internal_icfr`, `internal_management_reporting`, `internal_memos`, `internal_par`, `internal_process_and_controls`, `internal_wiki`).

## Subagent Structure

A typical subagent directory (e.g., `internal_capm/`) contains:

*   `__init__.py`: Marks the directory as a Python package.
*   `subagent.py`: Contains the primary logic for querying the specific data source, including connection handling, query formulation, and result processing.
*   `*_prompt.py` (Optional): Files defining specific prompts used by the subagent, often for tasks like selecting relevant sections (`catalog_selection_prompt.py`, `section_selection_prompt.py`) or synthesizing retrieved content (`content_synthesis_prompt.py`).

## Adding New Data Sources

To integrate a new data source:
1.  Create a new subdirectory within `database_subagents/` (e.g., `internal_new_source/`).
2.  Implement the `subagent.py` logic for connecting to and querying the source.
3.  Define any necessary prompts in separate files.
4.  Update the `database_router.py` and potentially the `agent_planner` to recognize and utilize the new subagent.
