Model Implementation

Model Name: IRIS (Intelligent Research and Information System)
Model ID: IRIS-001

Software / Coding

IRIS is implemented entirely in Python, with a modular architecture organized around specialized agents and knowledge sources. The system was developed as a Python package with clearly defined components, interfaces, and dependency management. This implementation approach provides flexibility for deployment while ensuring consistent functionality across environments.

The system architecture comprises four main components. The user-facing interface is implemented as a web application, initially using Python's Dash framework in Dataiku and later integrated into the Maven UI. The core model functionality is implemented as a Python API that processes queries through the agent framework. Database services use PostgreSQL with the pgvector extension, accessed through Python's psycopg2 library for document and vector storage. Integration with language models occurs through the RBC Lumina portal, with Python wrapper classes providing a consistent interface for all model interactions.

Each agent in the system is implemented as a Python module with standardized interfaces. The Router Agent analyzes query intent using Python's transformation functions for text processing. The Clarifier Agent implements context assessment logic using Python dictionaries for structured data organization. The Planner Agent uses Python's collections framework to manage database selection. Database Subagents leverage Python's object-oriented inheritance model for specialized retrieval methods. The Summarizer Agent employs Python's string manipulation capabilities for response formatting. All inter-agent communication uses Python's JSON handling for structured data exchange.

The knowledge store is implemented as a PostgreSQL database with four core tables accessed through Python's database connectors. Data flow follows a clearly defined path with standardized formats between components:

Information retrieval operates through three primary Python-implemented methodologies: Catalog Search - Small uses Python dictionary lookups, Catalog Search - Large implements hierarchical selection algorithms in Python, and Semantic Search - Large employs Python's numerical computing capabilities for vector comparisons. Python's concurrent.futures library enables parallel execution of database subagents when querying multiple sources.

Quality assurance is maintained through Python-specific tools. Code formatting uses Black with Python style guidelines. Linting employs Pylint configured for Python best practices. Manual testing is performed in both local development and Dataiku environments. This Python-centric implementation allows for consistent functionality across deployment environments while maintaining high code quality standards.

Implementation Testing

IRIS implementation testing follows a comprehensive multi-stage approach to ensure the system meets all business requirements and functions correctly in its target environment. Testing begins with the development team performing functional testing during local Python development to validate core functionality and behavior. This is followed by integration testing to verify that the assembled system functions as expected when all components operate together.

The implementation first deploys to Dataiku as a Python + Dash web application, where it undergoes functional testing by the development team to ensure all components work correctly in this environment. The Dataiku implementation is then made available to business users for user acceptance testing (UAT), which has provided valuable feedback on functionality, response quality, and usability. This feedback is incorporated into the system through an iterative development process, with each iteration undergoing regression testing to ensure existing functionality remains intact.

As the system transitions to the Maven environment, implementation testing follows a structured path with clearly defined stages and responsibilities:

  1. Development Environment Testing: After the Maven team implements the code in the development environment, the Digital Ecosystem & AI team conducts testing to verify all functionality works as expected, comparing results with the previously validated Dataiku implementation.
  2. UAT Environment Testing: Once verified in development, the system is pushed to the UAT SAI environment, where end users test the implementation to confirm it meets their business requirements.
  3. Production Testing: Following successful UAT, the final implementation undergoes formal post-implementation verification (PIV) testing in the production environment.

Throughout the testing process, the development team works closely with business users to ensure the implementation adequately addresses their requirements. Computational accuracy is verified by comparing results against previously validated outputs from the Dataiku implementation, confirming that the same inputs produce identical outputs across environments. Any discrepancies are investigated and resolved before proceeding to the next testing stage.

The implementation testing also includes performance validation, ensuring the system maintains acceptable response times under expected load conditions. Security testing verifies that access controls function correctly, with users only able to access databases appropriate to their role and permissions. Error handling is tested by deliberately introducing invalid inputs to confirm the system responds appropriately and provides meaningful error messages.

Final implementation testing will be completed in June when the system is fully integrated into the Maven production environment, confirming that the implementation consistently delivers the expected functionality while meeting all business and technical requirements.

Change Controls and Security

For ongoing maintenance and enhancements, a formal change process has been established. First, changes are developed and tested locally in the Python codebase. When validated, these changes are provided to the Maven team for implementation in the development environment, where the Digital Ecosystem & AI team verifies they're working as expected. The Maven team then pushes these changes to the UAT SAI environment for further testing before creating a formal change request to move the approved code to production. This structured approach ensures all changes are properly documented, tested, and approved.
